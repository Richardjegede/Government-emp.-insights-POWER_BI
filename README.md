Employement  insight Project using powerBI
I chose to work on this project by obtaining the raw dataset from Enterprise DNA and creating a compelling, interactive dashboard 
using Power BI.

Problem Statement
This Government employment data focuses on New Zealand comes from many different agencies and systems, making it hard to see a clear picture of the public sector workforce.
Information such as employee numbers, vacancies, roles, regions, and workforce trends is often reported in different ways and at different times. 
This makes it difficult for leaders and workforce planners to understand how government employment is changing over time.
The purpose of the Employment Insights dashboard is to help executives and HR teams easily track key workforce metrics within any selected time period, such as total headcount,open positions, turnover, and workforce distribution. 
However, working with this data is challenging because measures are not always consistent, reports are spread across multiple sources, and trends are not easy to visualise. 
For example,key employees indicator breakdown can be unclear.
Without a simple, centralised Power BI dashboard, it becomes harder to spot workforce risks, plan future staffing needs, and support informed decision-making. 
A unified dashboard is needed to bring employment data together, standardise key measures, and present clear, easy-to-understand insights for New Zealandâ€™s public sector leaders.

Solution
This Power BI dashboard brings employment data into one clear and easy-to-use view. 
It uses simple visuals to show workforce trends, changes over time, and key employment metrics such as headcount, vacancies, and turnover.
The dashboard helps managers and leaders quickly understand workforce performance and make better, data-driven decisions. 
It supports workforce planning by highlighting staffing levels, open roles, and employment trends.
By turning complex employment data into clear insights,
the dashboard improves transparency and helps organisations manage and plan their workforce more effectively.

Steps I Took In This Project
1. Planned the project using basic knowledge of the AIMS grid.
2. Imported employment-related data from Excel into Power BI.
3. Cleaned and transformed the data in Power Query by fixing errors, removing duplicates, and standardising fields such as names, regions, and date formats.
4. Built relationships between tables, including active and inactive relationships where required.
5. Completed the ETL (Extract, Transform, Load) process to prepare the data for analysis.
6. Created key employment measures and used them to build interactive visuals in Power BI.
7. Validated the data to confirm accuracy and reliability of the insights..

Major challenges and Customization made
